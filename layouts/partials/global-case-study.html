1. Creating the approach

When building software products, content is often distributed across the application. Some are UI-embedded, while some come from the backend and the database.

To ensure all content is delivered in the language of the user’s location, there needs to be a framework that supports retrieving the content in the appropriate language.

Our approach was to extract the strings of content, save their translations, and serve this translated version to the end user. To achieve this, we set up several steps in the frontend, backend, and database.

As part of the strategy to enhance user experience, we also made our solution more user-friendly by translating emails and supporting right-to-left languages.

2. Tackling the Frontend

Every software product typically communicates with its users through its UI. Our first challenge was translating the UI based on the user’s language preference.

The first step in tackling the content embedded in the UI was to move it from code to simple text files so that, down the line, introducing new languages can become as easy as replicating the flat file in a different language.

We used JSON files to separate code from the language embedded in it. We did this so that all the content is available in one place and a translation service can easily translate this content.

The next step we took involved generalizing all hardcoded words to use dynamically changing content based on the user’s location, for instance, displaying helplines based on the user’s locality.

Lastly, we introduced Crowdin, a third-party translation provider, to get translations for new languages. To extend this further, we integrated Crowdin with GitHub PR workflow so that the automated pipelines can handle the translations without manual intervention from a developer.

3. Next Step, the Backend 

In the backend, we maintained consistency and took similar steps to those in the frontend, ensuring no disruption to the user experience.

Since language is one of the most crucial aspects affecting the user’s experience, we needed a proper strategy to present various puzzle pieces such as web and database content.

The content served from the database was passed to the frontend directly without any translations. To change this, we configured the application to serve the translated data content to make this process seamless and used YAML files to store this translated data.

In the backend too, we configured the application to serve localized content based on the user’s language preferences. For database translations, we used Crowdin here as well.

4. Solution for Emails 

Next, we also wanted customers to receive emails in their target language.

In the case of emails, we integrated a third-party mail service provider, Iterable. Iterable offers smooth email translations thanks to its locale-based templates that automatically translate content based on the user's preference. For existing emails, we migrated them to use these easy-to-edit templates.

In the future, our client's marketing team can effortlessly launch events and email campaigns with these easy-to-edit templates.

For our client, this meant a better overall experience as well. Mails are now translated and can be modified without a dependency on the developer, removing a significant bottleneck for the client's marketing team.

5. Support for RTL Languages

While the application empowered users by enhancing language accessibility, the support for right-to-left (RTL) languages felt natural and intuitive.

The support for RTL languages was meticulously designed so that language barriers did not limit users. In fact, we identified the various elements and interactions to support RTL languages and incorporated them into the application.

Moreover, we also added support for bidirectional text and icons, which are essential for RTL languages. These features ensure that users can read and navigate through the application seamlessly without any confusion.

We also incorporated smart exceptions based on the content type. With these enhancements, users can access a broader range of content in their preferred language and enjoy an unparalleled experience while using the application.

Overall, these improvements transformed our solution into a robust tool that met the diverse needs of each end-user.
