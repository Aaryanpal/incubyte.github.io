1. Defining the Problem

Our client’s reach was limited within the US region for the first few years. They wanted to expand and reach a global audience by localizing the experience and providing high-quality care in numerous nations and languages.

Expanding such a product presented its own set of challenges that included the following:

  • Scalability of the product
  • Support for new countries
  • Support for new languages

In just three months, our team collaborated closely with the client to develop and deploy a framework to close the gap in global offerings, saving our clients both money and time.

2. Tackling the Frontend

Every software product typically communicates with its users through its UI.

Since language is one of the most crucial aspects affecting the user’s experience across the board, we needed a proper strategy to present various pieces of the puzzle, like emails, web content, and even database content.

Our first challenge was translating the UI based on the user’s language preference.

The solutions that we took included:

  • Separating languages from DB or UI to flat files
  • Separating code from the language embedded in it using JSON files
  • Generalizing hardcoded words and statements to use localized and dynamic versions
  • Introducing a third-party translation provider Crowdin
  • Integrating Crowdin with GitHub PR workflow to provide automated translations

3. Next Step, the Backend 

Regarding the application’s backend, we took similar steps to those in the frontend to ensure the user experience was never disrupted.

  • Identified and extracted required database content to YAML locale files
  • Configured the application to serve localized content based on the user’s language preference
  • Introduced third-party translation provider Crowdin for database translations
  • Integrated Crowdin with GitHub PR workflow to provide automated translations in the backend

4. Solution for Emails 

In the case of emails, we integrated a third-party mail service provider, Iterable. We also migrated existing emails to utilize easy-to-edit templates.

For our client, this meant a better overall experience as well. Mails are now translated and can be modified without a dependency on the developer, removing a significant bottleneck for the client’s marketing team.

4. Support for RTL Languages

While the application empowered users to access content in their selected language, the support for RTL language felt natural and intuitive.

For right-to-left languages, we identified the elements and interactions better to provide additional support to users. We added support for bidirectional text and icons. At the same time, we added smart exceptions based on the content type.
